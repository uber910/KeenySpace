# KeenySpace v0.1.0-alpha.1

**Image:** `ghcr.io/uber910/keenyspace:0.1.0-alpha.1`
**Status:** Alpha — single-org, docker-compose-only, production-ready for self-hosters comfortable with alpha software.

---

## What it is

KeenySpace is a self-hosted, open-source knowledge graph platform that keeps your data as plain Markdown files compatible with Obsidian. A server layer adds multi-user authentication, MCP access for LLM agents, and a server-side WAL-to-compile pipeline that turns append-only agent logs into structured knowledge pages — without ever requiring you to lock your content into a proprietary format. Losing the server means dropping back to single-user mode on the same files. There is no SaaS option.

## Highlights

**Single-process server (FastAPI + FastMCP)**
One ASGI process hosts the REST API at `/v1/api/*`, admin endpoints at `/v1/admin/*`, health at `/healthz` and `/readyz`, Prometheus metrics at `/metrics`, and the MCP transport at `/v1/mcp/` — all composed via FastMCP's `combine_lifespans()`. Single-worker uvicorn is the v1 baseline.

**11 Tier-1 MCP tools**
`read_page`, `append_log`, `list_workspaces`, `get_workspace_info`, `list_pages`, `search_workspace`, `get_instructions`, `list_blueprints`, `get_recent_changes`, `compile`, `compile_status` — covering agent read/write, workspace navigation, server-driven prompts, and compile control. All tools enforce the same auth middleware as HTTP endpoints.

**Server-side compile pipeline**
A per-workspace `CompileCoordinator` drives a pydantic-ai + instructor agent (default LLM provider is configurable; the reference provider is Claude via the `KEENYSPACE_COMPILE__PROVIDER` env var; OpenAI-compatible providers opt-in) that reads accumulated WAL entries and produces atomic page writes. Hard guardrails: step/token/time budgets, loop detection (same `(tool, args_hash)` 3x -> abort), idempotency hash check (identical WAL batch -> no-op), prompt-injection hardening (`<wal_entry>` delimiters, "data not instructions" system prompt, tool surface restricted to `read_page` + `search` only). APScheduler periodic backstop included.

**OIDC authentication + ks_live_* API keys**
External OIDC IdP (Authentik is the reference provider) for interactive login. `ks_live_*` API keys for programmatic MCP access — argon2id-hashed, plaintext shown once at creation, never-expiring by design (long-running MCP sessions vs short-lived OIDC tokens). Single Starlette `AuthMiddleware` covers HTTP and MCP surfaces identically. Group entry gate: `KEENYSPACE_AUTH__REQUIRED_GROUP` restricts OIDC login to members of a named group (API key holders bypass by design).

**Workspace lifecycle + blueprints**
Workspaces are Obsidian-compatible directories (UUID on disk, slug in API). Create from blueprint (default blueprint included), archive/unarchive, export (zip), import (markdown directory). `.obsidian/` excluded server-side from canon. Blueprint catalog via filesystem scan.

**Full `keenyspace` CLI + Claude Code hooks + daemon**
`init`, `login`, `logout`, `status`, `workspace list/create/use/pull/archive/register`, `ingest`, `query`, `lint`, `compile`, `doctor`, `backup`, `restore`. All Claude Code hooks (`session-start`, `session-end`, `pre-compact`, `post-compact`, `post-tool`) communicate with the background daemon over a Unix domain socket for sub-second hook latency. Post-compact context injection re-supplies workspace `CLAUDE.md` + root `index.md` + smart-selected concept pages after a compaction event. macOS launchd and Linux systemd-user templates included.

**Multi-arch Docker image**
`python:3.14.1-slim-bookworm` base, native amd64 and arm64 builds via GitHub Actions (ubuntu-24.04 + ubuntu-24.04-arm runners, no QEMU). `postgresql-client-17` included for backup/restore.

**Caddy and nginx reverse-proxy recipes**
`deploy/reverse-proxy/Caddyfile` (`flush_interval -1`, `transport { read_timeout 600s }`) and `deploy/reverse-proxy/nginx.conf` (`proxy_buffering off`, `proxy_read_timeout 600s`, `X-Accel-Buffering: no`) — both SSE/StreamableHTTP verified end-to-end.

**Opt-in Loki + Grafana + Prometheus observability**
`deploy/observability.yml` addon: Loki 3.4 + Promtail 3.4 (structlog JSON pipeline) + Grafana 12.4.4 with auto-provisioned datasources and a 5-panel KeenySpace metrics dashboard (compile runs by status, compile tokens, WAL appends by workspace, compile duration p95, Loki error/warning logs panel).

**Backup and restore**
`keenyspace backup` streams a gzip tarball (FS root + `pg_dump` + `manifest.json`) from the admin API. `keenyspace restore <archive>` replays it with version-compatibility check. `keenyspace doctor` validates FS/Postgres registry consistency and reports common failure modes.

## Install

Three commands to a running stack:

```
git clone https://github.com/uber910/KeenySpace
cd KeenySpace
./deploy/gen-secrets.sh        # generates deploy/.env with openssl-rand secrets (mode 600, never committed)
docker compose -f deploy/docker-compose.yml up -d
```

Wait for all services healthy:
```
curl http://localhost:8000/healthz
```

Then configure your OIDC IdP (Authentik is provisioned automatically via the included blueprint) and run `keenyspace init` to connect the CLI.

Full instructions: [docs/install.md](../install.md)

## Known limitations (alpha)

- **Single-org** — all authenticated users can access all workspaces. Per-workspace ACL (group-based authorization) is v1.5+.
- **Single-worker uvicorn** — multi-worker support requires `fcntl.flock` activation, APScheduler deduplication, and compile-agent isolation across workers; deferred to v1.5+.
- **No vector search** — navigation is wikilink traversal. Vector/semantic search is deferred to v2.
- **No real-time collaboration** — no cursors, no CRDT. This is a deliberate non-goal (see vision.md).
- **docker-compose only** — no Helm chart. Kubernetes deployment via Helm is v1.1+.
- **macOS Docker Desktop promtail limitation** — Promtail's `docker_sd_configs` scraping requires access to `/var/lib/docker/containers`, which is not accessible on macOS Docker Desktop. The observability addon works as expected on Linux hosts. macOS users can use stdout log tailing as an alternative.
- **`test-client` CI is red** — 4 pre-existing client test failures exist (2 ordering flakes in `test_config.py`, 2 sha256-prefix manifest mismatches in pull tests). The server test suite is 374/374 passing. Client test fixes are tracked for v0.1.1.
- **`KS_DRILL_API_KEY` must be provisioned** — the CI drill job requires this GitHub Actions secret to be set before the tag push triggers `drill.yml`.
- **`uv tool install` from local path untested** — the documented install path `uv tool install --from ./packages/client keenyspace` has not been end-to-end tested. The fallback `uv run keenyspace` (from the cloned repo) is verified working.
